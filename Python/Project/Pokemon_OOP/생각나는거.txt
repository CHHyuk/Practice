포켓몬 클래스(기본정보, 전투 넣고)를
내 유닛 / 적 유닛 클래스에 상속시켜서

내 유닛 클래스의 정보가 유저 클래스의 내 몬스터 리스트 각 인덱스에 들어가게끔
번호를 부여하고

팔거나 진화하거나 하면 인덱스 번호를 지우는식?

---------------------------------------

맵 시스템은

유저가 이동하는걸로 하고

맵 클래스를 만들어서

맵 클래스 안에 메뉴를 만들어야되나

아니면

맵마다 따로 메뉴판 만들어야되나

유저 정보에 맵 위치 변수를 넣어야되나

---------------------------------------

상점도 클래스로 나누는게 맞을텐데 아직 클래스 간에 정보교류 잘 모르는듯
(예를들면 유저 클래스를 이용해 만든 유저정보를 메뉴 클래스, 상점 클래스에 넣기
아니면 포켓몬 클래스를 이용해 만든 내 포켓몬 정보를 내 고유 정보로 만들어서 유저의 몬스터 리스트에 넣기)
